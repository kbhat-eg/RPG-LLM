## Main Program Logic

### Initial Setup

- The main program starts by setting key-related working fields and using a `setll` (Set Lower Limit) to position to the first record.

```rpg
c eval ra09l1_ikod = 0
c ra09l1_key setll ra09l1
```

- `ra09l1_key` is a key list (defined in the *inzsr subroutine).
- `read ra09l1` reads the first record.

### Processing Loop

- The loop continues as long as:
  - Not end-of-file (`not %eof`)
  - The field `raifir` (firm code in current record) matches the working firm's code.

```rpg
c read ra09l1
c dow not %eof and raifir = w_firm
   ...process...
c read ra09l1
c enddo
```

#### For Each Record

1. **Populate Data Structure:**

    ```rpg
    c eval d_kode = %triml(%editw(raikod : '   0'))
    c eval d_besk = raitxt
    c eval d_mail = raiema
    c eval d_mobi = raimob
    c eval w_parm = d_parm
    ```
    - `d_kode` gets the seller code, left trimmed and formatted.
    - `d_besk`, `d_mail`, `d_mobi` get relevant fields from the current record.
    - The parameter structure is moved to the string buffer `w_parm`.

2. **Call External Program:**

    ```rpg
    c call 'IT001R'
    c parm w_firm
    c parm w_prog
    c parm w_comd
    c parm w_parm
    c parm w_rpgp
    ```
    - Calls `IT001R`, passing the firm code, program name, command, parameter buffer, and program name again.

3. **Read Next Record**
    - Advances to the next record.
