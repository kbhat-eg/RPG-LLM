## Program Overview

This is a classic ILE RPG (RPG IV) program (member `Fo560R`) designed to work with a workstation display file (`Fo560d`) to **select gaps in number series**—as indicated by the description comments in Norwegian. Much of the code is focused on user interaction, validation, and parameter passing.

---

### **Header Specifications**

```rpg
h option(*nodebugio) datedit(*dmy)
```
- `*NODEBUGIO`: Disables some extra debug information for I/O operations, often for performance.
- `DATEDIT(*DMY)`: Default date format is day-month-year.

---

### **Files Declaration**

```rpg
fFo560d    cf   e             workstn
```
- Declares a display file (`Fo560d`) for communication with a workstation.

---

### **Variables and Data Structures**

#### **Parameter Variables**
```rpg
d p_aarr          s              4  0
d p_a1fr          s              8  0
d p_a1ti          s              8  0
d p_pf03          s              1  0
```
- `p_aarr`: Series year (or similar grouping), 4 digits.
- `p_a1fr`, `p_a1ti`: "From" and "To" numbers of the series, 8 digits each.
- `p_pf03`: Flag for F3 function key pressed.

#### **Local Data Area Structures**
```rpg
d                uds
d l_user                911    920
d l_firm                944    946  0
d l_fnav                951    980
```
- Maps sections of the RPG program's User Data Space (UDS) to variables, for user/firm metadata.

---

### **Main Program Logic**

#### **Display and Input**
```rpg
c     nyvis         tag
c                   exfmt     a1bld
```
- Tag `nyvis`: Used for looping back to input.
- `exfmt a1bld`: Display/input fields via display file format `a1bld`.

#### **Function Key Handling**
```rpg
c                   if        *inkc = '1'
c                   eval      p_pf03  = 1
c                   goto      avslutt
c                   endif
```
- Checks if the F3 (Exit) key was pressed (`*INKC` is a standard IBM indicator for function keys).
- Sets the parameter to signal F3, then exits.

#### **Validation: Year Must Be Entered**
```rpg
c                   if        a1aarr = *zeros
c                   eval      *in31 = '1'
c                   goto      nyvis
c                   else
c                   eval      *in31 = '0'
c                   endif
```
- If the year/series field is zero, sets indicator 31 to flag an error on the screen, and loops for new input.

#### **Validation: From Number ≤ To Number**
```rpg
c                   if        a1efaf > a1efat
c                   eval      *in32 = '1'
c                   goto      nyvis
c                   else
c                   eval      *in32 = '0'
c                   endif
```
- Validates that the "from" number is not greater than the "to" number. If error, sets indicator 32 and prompts for input again.

#### **Pass Back Values**
```rpg
c                   eval      p_aarr = a1aarr
c                   eval      p_a1fr = a1efaf
c                   eval      p_a1ti = a1efat
```
- Stores input values into the corresponding parameters for use by calling programs.

#### **Program End / Cleanup**
```rpg
c     avslutt       tag
c                   eval      *inlr = *on
c                   return
```
- Sets *INLR (Last Record Indicator) to `*ON` to close files and end program.

---

### **Initialization Subroutine**

```rpg
c     *inzsr        begsr
c     *entry        plist
c                   parm                    p_aarr
c                   parm                    p_a1fr
c                   parm                    p_a1ti
c                   parm                    p_pf03
c                   endsr
```
- RPG's special *INZSR subroutine for initialization.
- Defines the parameter list (`plist`) to accept the 4 parameters from the calling program.

---

## **Summary**

This program presents a display for entering/selecting a number series (and year), validates the input, and returns the results to the caller via parameters. 
- **Input validation** is done for both required fields and allowed ranges.
- **Error signaling** is performed through display file indicators.
- **Exit protocol and parameter passing** are handled cleanly.

**Use cases**: Data entry screens where the user must specify a valid number range within a series/year, with built-in validation and error messaging.
