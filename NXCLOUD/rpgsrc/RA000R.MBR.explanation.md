# RPG Source Code Explanation: Maintenance of Status Codes Common Program

This program is written in the traditional (fixed-format) ILE RPG and is intended as a "menu handler" for maintenance programs related to various status codes in a Norwegian accounting or ERP system. It allows users to select which type of status code they want to maintain and then calls the corresponding maintenance program.

## Header and Documentation

- **/TITLE** – Title for the program: "Vedlikehold statuskoder fellesprogram" ("Maintenance status codes common program").
- **h specification** – `datedit(*dmy)` sets default date format (day/month/year). `option(*nodebugio)` disables Debug I/O optimization for performance.

### Program Meta-Information

The comments at the top describe:
- System, program name (`RA000R`)
- Change log and comments (in Norwegian)
- Explanation of *indicators* and their usage, e.g., function keys (KA=F1, KC=F3), program flow, error handling, file access, etc.

## Main Logic Flow

### Initialization

Starts with a named tag (`dummy`). The program then moves to the "choose code type" section.

### Code Type Menu Loop

```rpg
c     a2taga        tag
* Nullstille
c                   move      00            a2valg
* Bare data
c     a2tagb        tag
c                   exfmt     a2bld
```
- **a2valg**: Variable cleared before the menu.
- **exfmt a2bld**: Display screen A2BLD (the selection screen).
  
### Exit Handling

```rpg
c     *inkc         cabeq     *on           xslutt
```
- If indicator KC (F3=Exit) is on, branch to program end.

### Input Validation

Checks if the selected code type (`a2valg`) is a valid value (00–09).

```rpg
c                   if        a2valg <> 00 and
c                             a2valg <> 01 and
... up to ...
c                             a2valg <> 09
c                   move      *on           *in31
c                   goto      a2tagb
c                   endif
```
- If invalid, set indicator 31 (error), and redisplay the menu.

### Program Dispatch

For each valid selection (00–09), call the corresponding maintenance program:

- 00 – Firma (Company): Call 'AA008R'
- 01 – Regnskapskoder (Accounting codes): Call 'RA001R'
- 02 – OCR-koder: Call 'RA002R'
- 03 – Reskontrokoder: Call 'RA003R'
- 04 – Sist brukte nummer (Last used number): Call 'RA004R' (with parameters)
- 05 – Betalingsvarsel (Payment notice): Call 'RA005R'
- 06 – Rentenota (Interest statement): Call 'RA006R'
- 07 – Registreringsbilde (Registration screen): Call 'RA007R'
- 08 – E-handelsinformasjon (E-commerce info): Call 'RA008R'
- 09 – Factoringinformasjon: Call 'RA009R'

After each call, the program returns to the menu for another possible selection.

Special for option 04: `RA004R` is called with parameters `w_firm` and `wpk04n`.

### Program Termination

```rpg
c     xslutt        tag
c                   eval      *inlr = *on
c                   return
```
- Tag for program end; sets the Last Record indicator to close files and exit.

- **Purpose:** Presents a menu for users to select which kind of status code they want to maintain, then calls the corresponding program.
- **Design:** Loop around a selection screen; dispatch to sub-programs; validate input; support for keyboard function keys.
